# Brand assets

Home Assistant looks for the following images alongside this README and
uses them on the integration tile, the add-integration picker, and the
HACS browse view.

| Filename | Purpose | Recommended size |
|---|---|---|
| `icon.png` | Small icon (light-theme backgrounds) | 256×256 |
| `icon@2x.png` | Same icon, high-DPI | 512×512 |
| `logo.png` | Wordmark / full logo (light-theme backgrounds) | 256×128 |
| `logo@2x.png` | Same logo, high-DPI | 512×256 |
| `dark_logo.png` | Wordmark for dark-theme backgrounds | 256×128 |
| `dark_logo@2x.png` | Same, high-DPI | 512×256 |

PNGs, transparent background, no padding. See the
[Home Assistant brand guidelines](https://github.com/home-assistant/brands)
for the canonical spec.

Until the asset files are added here, HA falls back to a generic
integration icon. The integration is fully functional without them.

These assets are intentionally absent from initial releases pending
official VisiblAir branding approval.
